# AI伙伴协作指南

## 总工程师：Kilo Code

### 协作原则
作为跨客户端AI协作系统，我们需要确保每个AI伙伴都能无缝继续其他伙伴的工作。以下是协作的基本要求：

### 1. 实时进度更新
- **必须**在每次完成一个小阶段的更改后，立即更新 `Idle/copilot.md`
- **必须**详细记录当前进度、计划、所有理解和必要的上下文
- **必须**使用清晰的中文描述，包含技术细节和决策理由

### 2. 分支管理策略
- **必须**将各个小阶段和可以归为一类的更改分别分步提交
- **必须**将不同的大分类功能分别提交到不同的分支上
  - 例如：用户界面和核心功能分为两个分支
  - 安全功能和性能优化分为不同分支
  - 测试相关和文档相关分为独立分支

### 3. 状态同步机制
- **必须**在 `Idle/copilot.md` 中维护完整的项目状态
- **必须**记录当前正在处理的任务和已完成的任务
- **必须**标注每个任务的状态（进行中/已完成/待处理）

### 4. 上下文传递
- **必须**详细说明代码变更的原因和影响
- **必须**记录遇到的技术挑战和解决方案
- **必须**标注重要的设计决策和架构考虑

### 5. 质量保证
- **必须**在提交前进行必要的测试
- **必须**记录测试结果和发现的问题
- **必须**标注需要其他伙伴关注的潜在风险

### 6. 沟通规范
- **必须**使用友好的语气和表情符号，保持积极协作氛围
- **必须**及时响应其他伙伴的更新和问题
- **必须**在遇到困难时主动寻求帮助
